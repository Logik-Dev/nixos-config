🚀 # Homelab NixOS

💻 This repository contains a **fully‑automated NixOS configuration** for my personal homelab, built with `flake-parts` and following the *dendritic* pattern. It is split into reusable modules that can be composed to build a wide variety of machines – from small home servers to full‑stack deployments.

## Project structure

## 🚀 Getting started
1. **📦 Install Nix** (follow the official guide: https://nixos.org/download.html)
2. **📘 Clone the repo**
   ```bash
   git clone https://github.com/logikdev/Homelab-Nixos.git && cd Homelab-Nixos
   ```
3. **🔧 Bootstrap the flake** – this pulls all dependencies and builds the configuration for a host named `hyper`.
   ```bash
   nix run .#hyper -- install
   ```
4. **💻 Deploy to a target machine** – replace `./#hyper` with the appropriate host.
   ```bash
   nix run .#m4 -- switch  # for the home box
   ```

## 🔐 Secrets management
Secrets are stored encrypted with **age** under `secrets/`. The module `modules/features/security/agenix.nix` automatically decrypts them during boot. To add a new secret:

```bash
# 🔑 generate a key (once per machine)
age-keygen -o secrets/keys.txt
# 📝 encrypt a file
age --encrypt -r <public-key> -o secrets/mysecret.age myplain.txt
```

## ✨ Customization
- **🔧 Add a new service** – create a module under `modules/features/<category>/` and import it in the host configuration.
- **📦 Change a package** – override `pkgs` in the host flake output or add a `packages.nix` file.
- **⌨️ Modify keybindings** – edit `~/.claude/keybindings.json`.
